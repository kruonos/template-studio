# Code of Conduct

Pretext Template Studio should be a useful place to discuss editor architecture, layout engines, document generation, exports, and local-first product work.

## Expected Behavior

- Be direct, specific, and respectful.
- Critique code, behavior, documentation, and tradeoffs rather than people.
- Assume contributors may be testing across different browsers, operating systems, email clients, and document tools.
- Keep issue reports actionable with steps, screenshots, exported files, or audit output when possible.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information or credentials.
- Repeated off-topic comments that make technical discussion harder.
- Abuse of issue templates, discussions, or pull requests.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject contributions; or block participants when behavior makes the project unsafe or unproductive.

For sensitive reports, use GitHub private vulnerability reporting when available, or contact the maintainer privately through the repository owner profile.
